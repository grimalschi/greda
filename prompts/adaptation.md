# Промпт генерации адаптации (graded reader на испанском)

Этот файл — шаблон для офлайн-этапа генерации контента (ТЗ §11). Контент создаётся
**вне приложения** и коммитится как готовые JSON. Приложение во время работы LLM не вызывает.

## Вход

- Исходный public domain текст (английский или испанский) — `source/source.txt`.
- Метаданные источника — `source/source-metadata.json`.
- Целевой уровень: один из `A2 | B1 | B2 | C1`.

## Задача

Создать **испанскую адаптацию** исходного произведения под конкретный уровень CEFR,
затем разбить её на предложения и дать естественный **русский перевод каждого предложения**.

### Правила адаптации

1. Это graded reader: главное — понятность уровня, а не буквальное сохранение стиля.
2. **Сокращение только мягкое.** Нельзя выкидывать целые сюжетные сцены.
   Можно упрощать длинные описания, сложные обороты, второстепенные детали.
3. Сохрани последовательность событий и ключевые образы.
4. Текст пишется **только на испанском**. Оригинал не вставляем.
5. Перевод — на уровне **предложений**, без пословного перевода, словаря, лемм, морфологии.
6. Русский перевод — естественный и литературный, не подстрочник.

### Ориентиры по уровням

- **A2** — короткие простые предложения; настоящее и простое прошедшее; базовая лексика;
  минимум придаточных.
- **B1** — связный текст, умеренные придаточные, более широкая бытовая лексика.
- **B2** — богаче лексика и синтаксис, идиоматичность, разные времена.
- **C1** — близко к литературному, сложные конструкции, тонкие оттенки смысла.

## Выход

Для каждой главы — файл `levels/<level>/chapter-XXX.json` строго по схеме
`schemas/chapter.schema.json`:

```json
{
  "schemaVersion": "1.0",
  "workId": "<id произведения>",
  "level": "<a2|b1|b2|c1>",
  "chapter": { "id": "chapter-001", "number": 1, "title": "Заголовок на испанском" },
  "paragraphs": [
    {
      "id": "para-001",
      "sentences": [
        { "id": "sent-001", "text": "…", "translationRu": "…" }
      ]
    }
  ]
}
```

И `levels/<level>/manifest.json` по `schemas/manifest.schema.json`.

### Требования к id

- `para-001`, `para-002`, … — сквозная нумерация абзацев в главе.
- `sent-001`, `sent-002`, … — сквозная нумерация предложений в пределах главы.
- `sentenceCount` в манифесте обязан совпадать с реальным числом предложений.

## Пайплайн (ТЗ §11)

1. Сохранить исходный текст → `source/source.txt`.
2. Очистить текст (убрать колонтитулы, артефакты OCR).
3. Разбить на главы/абзацы.
4. Для каждого уровня A2/B1/B2/C1 создать испанскую адаптацию.
5. Разбить адаптацию на предложения.
6. Для каждого предложения — естественный русский перевод.
7. Сохранить JSON по схемам.
8. `npm run validate`.
9. Закоммитить.

После добавления уровня обновить `work.json` (`levels.<level>.available = true`,
`chapterCount`) и `catalog.json` (`availableLevels`).
